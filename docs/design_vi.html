1. Tổng quan

Draox Server là một server socket đa giao thức viết bằng Rust, được thiết kế theo kiến trúc plugin-based (lấy cảm hứng từ VS Code). Server quản lý kết nối client qua nhiều giao thức (TCP, UDP, WebSocket, HTTP/HTTPS) và cho phép mở rộng chức năng thông qua hệ thống plugin hybrid: Built-in (Rust crate biên dịch cùng) và External (WASM sandbox qua wasmtime).

Dự án được tổ chức dạng Cargo workspace module hóa với 21 crate chuyên biệt trải dài 7 tầng kiến trúc, bao gồm socket server đa giao thức, bảo vệ traffic (chống spam, DDoS), quản lý kết nối server-authoritative hỗ trợ nhiều kết nối đồng thời mỗi client, lưu trữ database (SQL + NoSQL), bộ nhớ đệm (Redis + in-memory), nhật ký hoạt động, thanh toán, và REST API quản trị với ~72 endpoint cùng 5 WebSocket stream. Marketplace tích hợp cho phép khám phá, cài đặt và quản lý plugin.

Hệ thống Plugin

Kiến trúc plugin hybrid lấy cảm hứng từ VS Code. Hỗ trợ Built-in (Rust) và External (WASM sandbox qua wasmtime) với marketplace để khám phá và phân phối plugin.

Hiệu suất cao

Xây dựng trên Tokio async runtime với zero-copy I/O, đồng thời theo task-per-connection, và cấu trúc dữ liệu lock-free để đạt thông lượng tối đa.

Server-Authoritative

Hỗ trợ nhiều kết nối đồng thời mỗi client với session binding, failover tự động, state reconciliation, và heartbeat đa kết nối.

Sẵn sàng cho Production

TLS/mTLS, xác thực JWT, giới hạn tốc độ, structured logging, graceful shutdown, và WASM sandbox cách ly plugin bên ngoài cho môi trường production.

Bảo vệ Traffic

Chống spam kết nối, DDoS, tự động ban/mở ban IP, phân tích hành vi bất thường, điều chỉnh ngưỡng thích ứng.

5. Traffic Guard — Chống Spam & DDoS

Crate traffic-guard cung cấp giải pháp chống spam và DDoS tập trung. Nằm giữa socket-server và connection-manager trong pipeline xử lý. Lọc traffic độc hại ở tầng mạng trước khi tiêu tốn tài nguyên server.

A. Luồng xử lý Traffic

  Client ──TCP/UDP/WS/HTTP──→ socket-server ──→ traffic-guard ──→ connection-manager
                                                     │
                                                     ├── ✅ CHO PHÉP → chuyển đến connection-manager
                                                     ├── ⛔ CHẶN → hủy kết nối, ghi log
                                                     └── ⏳ GIỚI HẠN → trì hoãn, backpressure

    async fn deactivate(&mut self) -> Result<(), PluginError>;

    /// Xử lý command từ client hoặc admin
    async fn execute_command(
        &self,
        command: &str,
        params: serde_json::Value,

    ) -> Result<serde_json::Value, PluginError>;

    /// Xử lý sự kiện hệ thống
    async fn on_event(&self, event: &SystemEvent) -> Result<(), PluginError>;
}

13. Plugin — Nhắn tin Tức thời (plugin-messaging)

Plugin tích hợp (built-in) cung cấp hệ thống nhắn tin tức thời với các loại tin nhắn: trực tiếp (1:1), kênh (1:N), phát sóng (Admin → Tất cả), và hệ thống.

Sơ đồ Loại Tin nhắn

  Loại tin nhắn:

  1. Trực tiếp (1:1)          User A ──────▶ User B
  2. Kênh (1:N)               User A ──────▶ Kênh #general ──▶ [User B, C, D, ...]
  3. Phát sóng (Admin→All)    Admin  ──────▶ ★ Tất cả users online
  4. Hệ thống (System)        Server ──────▶ ★ Tất cả hoặc nhóm cụ thể

Message Envelope (JSON)

{
  "id": "msg-uuid-001",
  "type": "direct",                // direct | channel | broadcast | system
  "from": "user-abc",
  "to": "user-xyz",               // user ID, channel ID, hoặc "*" cho broadcast
  "content": {
    "text": "Xin chào!",
    "attachments": [],
    "metadata": {}
  },
  "timestamp": "2026-04-13T10:30:00Z",
  "status": "sent",               // sent | delivered | read
  "reply_to": null,               // ID tin nhắn trả lời (tùy chọn)
  "ttl": null                     // Thời gian sống (giây, tùy chọn)
}

1. WASM Runtime (wasmtime) — 10 mục

Mục đích: Cho phép load và chạy plugin bên thứ 3 được viết bằng bất kỳ ngôn ngữ nào (Rust, C, Go, AssemblyScript…) biên dịch sang WebAssembly, chạy trong sandbox cách ly với giới hạn bộ nhớ và CPU.

Tại sao chưa triển khai: wasmtime v29 (đã có trong workspace deps) là một runtime engine thực. Nó biên dịch .wasm modules, tạo Store cho mỗi plugin, cấp phát bộ nhớ ảo, và đo CPU fuel. Mock không có giá trị — toàn bộ ý nghĩa nằm ở sự cách ly thực sự (memory fences, WASI sandboxing, CPU fuel limiting). Một mock sẽ không chứng minh được plugin độc hại không thể thoát khỏi sandbox.

Cần gì để triển khai:

  • Không cần server bên ngoài, nhưng cần binary .wasm thực để test
  • Định nghĩa WIT (WebAssembly Interface Types) — bản hợp đồng API giữa host và plugin
  • Toolchain wit-bindgen để generate binding code cho cả hai phía
  • 10 mục: khởi tạo engine, biên dịch/cache module, hỗ trợ WASI, WIT API boundary, giới hạn bộ nhớ per-plugin, timeout per-call, CPU fuel limiting, Store isolation per-plugin, xác minh sandbox, định nghĩa WIT

Tại sao chưa triển khai: Cần file cơ sở dữ liệu MaxMind GeoLite2 (~60 MB binary). File này yêu cầu đăng ký tài khoản MaxMind (miễn phí) và tải về. Crate maxminddb đọc file này để tra cứu IP → Quốc gia/Thành phố.

Cần gì: Tài khoản MaxMind + file GeoLite2-Country.mmdb đặt tại đường dẫn trong config.

6. OpenAPI / Swagger UI — 1 mục

Mục đích: Tự động sinh tài liệu API tương tác từ code, phục vụ tại /swagger-ui.

Tại sao chưa triển khai: utoipa v5 + utoipa-swagger-ui v9 (đã có trong deps) yêu cầu thêm #[utoipa::path(...)] macro annotation vào mỗi route handler (~50+ endpoints). Đây là công việc cơ học lớn nhưng không thêm logic mới. Ngoài ra utoipa-swagger-ui bundle ~2 MB static assets, tăng kích thước binary.

Cần gì: Không cần dịch vụ bên ngoài — chỉ cần thời gian annotate tất cả endpoints với request/response schemas. Tất cả response types đã implement Serialize.

21. Tổng kết

Draox Server v2.2 là một server socket đa giao thức viết bằng Rust với kiến trúc plugin-based linh hoạt. Hệ thống được thiết kế module hóa với 21 crate chuyên biệt trải dài 7 tầng kiến trúc, cho phép phát triển, kiểm thử và triển khai độc lập từng thành phần.

Điểm nổi bật chính của kiến trúc:

Cargo Workspace Module hóa

21 crate chuyên biệt trải dài 7 tầng kiến trúc, mỗi crate có ranh giới rõ ràng, cho phép phát triển, kiểm thử và triển khai độc lập từng thành phần.

Hệ thống Plugin Hybrid

Kết hợp Built-in (Rust native) cho hiệu suất tối đa và External (WASM sandbox) cho khả năng mở rộng an toàn từ cộng đồng. Marketplace tích hợp hỗ trợ khám phá và phân phối.

Server-Authoritative

Mô hình multi-connection cho phép mỗi client duy trì nhiều kết nối với vai trò khác nhau. Server kiểm soát toàn bộ trạng thái với failover và state reconciliation tự động.

Đa Giao thức

Socket server hỗ trợ TCP, UDP, WebSocket, HTTP/HTTPS với TLS/mTLS, cung cấp nền tảng vững chắc cho mọi loại ứng dụng kết nối.

Bảo vệ Traffic

Chống spam và DDoS tập trung với hệ thống uy tín IP, tự động ban/mở ban, phân tích hành vi, và điều chỉnh ngưỡng thích ứng dựa trên tải server.

Hạ tầng Production qua Docker

Stack 4 service: PostgreSQL 16, Redis 7, draox-server, và Admin Dashboard (nginx SPA). Multi-stage builds, health check phụ thuộc, giới hạn tài nguyên.

Định danh & Xác thực

Plugin-identity cung cấp JWT, OAuth2, API Key, RBAC đa cấp, quản lý vai trò & quyền, hồ sơ người dùng và danh sách bạn bè.

Phân cụm & Phân tán

Plugin-cluster hỗ trợ triển khai đa node với đồng bộ trạng thái, cân bằng tải thích ứng, phân vùng session và phục hồi node tự động.

Dashboard Admin

Giao diện web UmiJS 4 + Ant Design Pro 6 với 18 trang quản lý, dark theme, real-time chart, và proxy nginx đến Admin API.
