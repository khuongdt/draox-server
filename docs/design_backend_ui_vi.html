1. Tổng quan

Admin Dashboard của Draox Server là giao diện quản trị dựa trên web, được xây dựng bằng Ant Design Pro 6 — framework quản trị doanh nghiệp hàng đầu dựa trên React 18 và Ant Design 5. Dashboard cung cấp khả năng giám sát thời gian thực, quản lý kết nối/phiên/plugin, kiểm soát bảo mật lưu lượng, và quản lý marketplace plugin thông qua giao diện tối (Dark Theme) nhất quán với bảng màu thiết kế server.

Toàn bộ dữ liệu trực tiếp được cập nhật thông qua 5 luồng WebSocket kết nối đến Admin API (cổng 9100), đảm bảo thông tin luôn mới nhất mà không cần làm mới trang. Hệ thống kiểm soát truy cập dựa trên vai trò (RBAC) với 3 vai trò phân quyền rõ ràng, và hỗ trợ đa ngôn ngữ tiếng Anh và tiếng Việt.

⚙ Ant Design Pro 6

React 18 + UmiJS 4 + Ant Design 5. Tích hợp sẵn ProLayout, ProTable, ProForm, ProCard. Framework quản trị doanh nghiệp được sử dụng rộng rãi với hệ sinh thái component phong phú và hỗ trợ TypeScript đầy đủ.

🎨 Giao diện tối (Dark Theme)

Ánh xạ token tùy chỉnh từ bảng màu thiết kế server. Nhất quán với báo cáo thiết kế kiến trúc. Bảng màu: Navy (#1a1a2e) + Cam (#e05d10) + Xanh lá (#53c28b).

⚡ Cập nhật thời gian thực

Tích hợp WebSocket cho 5 luồng trực tiếp: kết nối, phiên, plugin, metrics, sự kiện. Tự động kết nối lại qua reconnecting-websocket. Dữ liệu metrics, sự kiện, cảnh báo bảo vệ được cập nhật liên tục.

🌎 Đa ngôn ngữ & RBAC

Hỗ trợ i18n cho Tiếng Anh (en-US) và Tiếng Việt (vi-VN). Plugin locale của UmiJS. Kiểm soát truy cập dựa trên vai trò (RBAC) với 3 vai trò: Admin, Operator, Viewer.

Thanh sức khỏe hệ thống (HealthBar)

Header phụ nằm dưới header chính, hiển thị trạng thái sức khỏe thời gian thực của từng component thông qua các chấm màu có nhãn. Dữ liệu lấy từ endpoint GET /api/health và tự động làm mới mỗi 30 giây.

Màu sắc trạng thái sức khỏe:

  • ● Healthy — Component hoạt động bình thường
  • ● Degraded — Component hoạt động nhưng có vấn đề
  • ● Unhealthy — Component không hoạt động
  • ● Unknown — Không thể xác định trạng thái

Thẻ thống kê (StatisticCard)

Thẻ Giá trị hiển thị Nguồn dữ liệu Xu hướng

Biểu đồ Dashboard

Biểu đồ vùng (Area) — Lưu lượng mạng

  • Thư viện: @ant-design/charts > Area
  • Dữ liệu: bytes_sent và bytes_received theo thời gian
  • Khung thời gian: 1 giờ gần nhất (60 điểm)
  • Cập nhật: mỗi 5 giây qua /ws/metrics
  • Màu: Cam (#e05d10) và Xanh lá (#53c28b)

Biểu đồ tròn (Pie) — Phân bố giao thức

  • Thư viện: @ant-design/charts > Pie
  • Dữ liệu: Số kết nối theo giao thức (TCP/UDP/WS/HTTP)
  • Cập nhật: mỗi 30 giây
  • Hiển thị: Tỷ lệ phần trăm trên mỗi slice

Biểu đồ đường (Line) — Kết nối theo thời gian

  • Thư viện: @ant-design/charts > Line
  • Dữ liệu: Tổng kết nối theo phút (real-time)
  • Bộ đệm vòng: 60 phút dữ liệu lịch sử
  • Cập nhật: push từ /ws/metrics

EventTimeline — Sự kiện gần nhất

  • Hiển thị 20 sự kiện gần nhất
  • Cuộn tự động khi có sự kiện mới
  • Màu theo loại: info=xanh, warn=vàng, error=đỏ
  • Cập nhật qua /ws/events

Code mẫu Dashboard

// pages/Dashboard/index.tsx
import { StatisticCard } from '@ant-design/pro-components';
import { Area, Pie, Line } from '@ant-design/charts';
import { useModel } from '@umijs/max';
import { EventTimeline } from '@/components/EventTimeline';
import { HealthStatusDot } from '@/components/HealthStatusDot';

const Dashboard: React.FC = () => {
  const { metrics, events } = useModel('metrics');

  return (
    <>
      {/* Health bar */}
      <HealthBar components={['tcp','udp','websocket','http','database','cache']} />

      {/* Stats row */}
      <ProCard ghost gutter={[16, 16]}>
        <StatisticCard statistic={{ title: 'Kết nối hoạt động',
          value: metrics.activeConnections, trend: 'up' }} />
        <StatisticCard statistic={{ title: 'Phiên hoạt động',
          value: metrics.activeSessions }} />
        <StatisticCard statistic={{ title: 'Plugin đã đăng ký',
          value: metrics.registeredPlugins }} />
        <StatisticCard statistic={{ title: 'Tỷ lệ lỗi',
          value: metrics.errorRate, suffix: '%',
          valueStyle: { color: metrics.errorRate > 1 ? '#ff4d4f' : '#53c28b' } }} />
      </ProCard>

      {/* Charts */}
      <ProCard ghost gutter={[16, 16]}>
        <Area data={metrics.traffic} xField="time"
          yField="value" seriesField="type" />
        <Pie data={metrics.protocols} angleField="count"
          colorField="protocol" />
      </ProCard>
    </>
  );
};

Kiến trúc WsManager

// utils/wsManager.ts — Singleton quản lý 5 WebSocket streams
import ReconnectingWebSocket from 'reconnecting-websocket';

type StreamName = 'connections' | 'sessions' | 'plugins' | 'metrics' | 'events';
type MessageHandler = (data: unknown) => void;

class WsManager {
  private sockets: Map<StreamName, ReconnectingWebSocket> = new Map();
  private handlers: Map<StreamName, Set<MessageHandler>> = new Map();
  private baseUrl: string;

  constructor(baseUrl: string) {
    this.baseUrl = baseUrl.replace(/^http/, 'ws');
  }

  connect(stream: StreamName, token: string): void {
    if (this.sockets.has(stream)) return;
    const ws = new ReconnectingWebSocket(
      `${this.baseUrl}/ws/${stream}?token=${token}`,
      [],
      { maxReconnectionDelay: 10000, reconnectionDelayGrowFactor: 1.3 }
    );
    ws.onmessage = (event) => {
      const data = JSON.parse(event.data);
      this.handlers.get(stream)?.forEach(h => h(data));
    };
    this.sockets.set(stream, ws);
  }

  subscribe(stream: StreamName, handler: MessageHandler): () => void {
    if (!this.handlers.has(stream)) this.handlers.set(stream, new Set());
    this.handlers.get(stream)!.add(handler);
    return () => this.handlers.get(stream)?.delete(handler);  // Unsubscribe
  }

  disconnectAll(): void {
    this.sockets.forEach(ws => ws.close());
    this.sockets.clear();
  }
}

export const wsManager = new WsManager(process.env.ADMIN_API_URL || 'http://localhost:9100');

  // Add JWT token to all requests
  requestInterceptors: [

      if (token) config.headers!['Authorization'] = `Bearer ${token}`;
      return config;
    },
  ],

  // Handle API envelope and errors
  responseInterceptors: [
    (response) => {
      const { data } = response;
      if (data.success === false) {
        notification.error({ message: data.error?.message || 'Lỗi không xác định' });
        throw new Error(data.error?.message);
      }
      return response;
    },
  ],

  errorConfig: {
    errorHandler: (error) => {
      if (error.response?.status === 401) {
        history.push('/login');
      }

  },

export default function access(initialState: { currentUser?: API.User }) {
  const { currentUser } = initialState ?? {};

    canManagePlugins: role !== 'viewer',
    canManageTraffic: role !== 'viewer',

    canFlushCache: role === 'admin',
    canExportAuditLog: role === 'admin',
    canManageRoutes: role === 'admin',
    canManageBilling: role === 'admin',
  };
}

// components/BytesDisplay/index.tsx
interface BytesDisplayProps {
  bytes: number;
  precision?: number;
  colored?: boolean;  // Green for low, yellow for medium, red for high
}

const UNITS = ['B', 'KB', 'MB', 'GB', 'TB'];

export const BytesDisplay: React.FC<BytesDisplayProps> = ({
  bytes, precision = 2, colored = false
}) => {
  const unitIndex = bytes === 0 ? 0 : Math.floor(Math.log(bytes) / Math.log(1024));
  const value = (bytes / Math.pow(1024, unitIndex)).toFixed(precision);
  const unit = UNITS[Math.min(unitIndex, UNITS.length - 1)];

  return (
    <span style={colored ? { color: unitIndex >= 3 ? '#ff4d4f' : '#53c28b' } : {}}>
      {value} {unit}
    </span>
  );
};

      title: intl.formatMessage({ id: 'connections.table.status' }),
      dataIndex: 'status',
      render: (s: string) => <ConnectionStatusTag status={s} />,
    },
  ];

  return <ProTable columns={columns} ... />;
};

Selector ngôn ngữ

SelectLang component trong header ProLayout cho phép chuyển đổi ngôn ngữ. Sử dụng setLocale('vi-VN') từ @umijs/max. Trang tự động render lại với ngôn ngữ mới mà không cần tải lại.

18. Các giai đoạn triển khai

Dự án được chia thành 5 giai đoạn triển khai tuần tự, mỗi giai đoạn cung cấp giá trị sử dụng ngay từ khi hoàn thành.

1
Giai đoạn 1: Nền tảng

Khởi tạo dự án Ant Design Pro 6 + UmiJS 4. Cấu hình dark theme token. Triển khai xác thực JWT (đăng nhập, đăng xuất, refresh). Tạo 13 file dịch vụ API. Cấu hình routes với RBAC. Triển khai WsManager singleton. Tạo 11 component cơ bản. Cấu hình proxy phát triển.

2
Giai đoạn 2: Trang chính

Dashboard (StatisticCard + biểu đồ tĩnh + HealthBar). Trang Kết nối (ProTable + ngắt kết nối). Trang Phiên (ProTable + hủy/drain). Trang Plugin (ProTable + hành động vòng đời). Trang Traffic Guard (3 tab + reputation). Trang Cấu hình (ProForm nhóm + Monaco Editor). Trang Cache (stats + quản lý keys).

3
Giai đoạn 3: Thời gian thực

Kết nối 5 luồng WebSocket vào WsManager. Dashboard cập nhật thời gian thực. Trang Kết nối tự động refresh từ /ws/connections. Models UmiJS cho metrics và events. WebSocketIndicator trong header. Xử lý mất kết nối và kết nối lại. EventStream page với virtual scroll.

4
Giai đoạn 4: Nâng cao

Trang Metrics đầy đủ (6 thẻ + 4 biểu đồ + tóm tắt 24h). Marketplace (duyệt + chi tiết + cài đặt + xuất bản). Trang Billing (gói + biểu đồ + lịch sử). Nhật ký kiểm toán (ProTable + lọc nâng cao). Quản lý Routes. Chi tiết kết nối và phiên (trang con).

5
Giai đoạn 5: Hoàn thiện

Hoàn thành bản dịch tiếng Việt (~165 chuỗi). Kiểm tra RBAC toàn bộ trang với 3 vai trò. Responsive design cho màn hình nhỏ hơn. Trạng thái tải (Skeleton) và xử lý lỗi (Empty, Error boundary). Kiểm tra hiệu suất với React DevTools Profiler. Tài liệu Storybook cho 11 component. Unit test cho các service và utility.

Điểm mạnh chính của thiết kế

⚡ Hiệu suất cao

  • Virtual scroll cho danh sách lớn (EventStream, danh sách logs)
  • Bộ đệm vòng giới hạn dữ liệu WS (10k sự kiện, 60 điểm metrics)
  • Debounce và throttle cho tìm kiếm và cập nhật
  • React.memo và useMemo cho component tốn kém

🔓 Bảo mật nhiều lớp

  • JWT token với tự động refresh trước khi hết hạn
  • RBAC chi tiết: ẩn/hiện button theo vai trò
  • Server-side enforcement: API từ chối nếu không có quyền
  • Audit log ghi lại tất cả hành động quản trị

🌞 Độ tin cậy cao

  • WS tự động kết nối lại với exponential backoff
  • Hiển thị dữ liệu "stale" khi mất kết nối WS
  • Error boundary React ngăn crash toàn trang
  • Popconfirm cho tất cả hành động nguy hiểm

🌎 Trải nghiệm người dùng

  • Dark theme nhất quán với bảng màu Draox Server
  • Responsive design hỗ trợ màn hình từ 768px
  • Skeleton loading trong khi tải dữ liệu
  • Thông báo lỗi rõ ràng, có hướng dẫn khắc phục
