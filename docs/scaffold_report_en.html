2. Technology Stack

The frontend is built on a curated enterprise React stack. Each technology was chosen for its production track record, dark theme support, and compatibility with the Draox Server admin API requirements.

React React 18

Latest React with concurrent rendering features. The scaffold uses hooks-based architecture throughout — useState, useEffect, useRef, and custom hooks. No class components. Concurrent mode enables smooth UI updates for real-time metrics streams.

AntD Ant Design 5

Enterprise UI component library with first-class dark theme support via the darkAlgorithm token. CSS variables-based theming allows runtime token overrides. Custom design tokens map the orange brand color to colorPrimary across all components.

UmiJS UmiJS 4 / @umijs/max

Enterprise React meta-framework providing: file-based routing with RBAC access control, global data flow via useModel, request plugin with interceptors, built-in internationalization, proxy configuration, and mock server — all via a single @umijs/max dependency.

Pro Ant Design Pro Components

High-level business components built on Ant Design 5:

  • ProLayout — Sidebar + header + breadcrumbs
  • ProTable — Data tables with search, filter, export
  • ProForm / StepsForm — Form management
  • ProDescriptions — Detail view layout
  • ProCard — Dashboard card containers

Charts @ant-design/charts

G2-based charting library with dark theme support. Used for Area charts (bandwidth), Line charts (metrics timeseries), Column charts (request rates), Pie charts (protocol distribution), and Gauge charts (IP reputation scores). All charts configured with dark backgrounds and orange accent colors.

TS TypeScript 5

Full type coverage across all source files with strict mode enabled. No any types used. Custom type definitions in src/typings.d.ts and src/services/typings.d.ts. Path aliases configured via @/* mapping to src/* for clean imports.

app.tsx — Runtime Config

  • getInitialState: Calls GET /api/auth/me on app load to hydrate currentUser. Redirects to /login on 401.
  • layout: Returns ProLayout config with logout action — clears localStorage token and redirects to /login.
  • request interceptors: Reads token from localStorage, injects Authorization: Bearer <token> header on every outbound request.
  • response interceptors: Unwraps { success, data, message } envelope — returns data to callers, shows error notification on success: false.

access.ts — Permission Evaluation

The access.ts function receives initialState (containing currentUser) and returns a flat object of boolean flags. UmiJS route access fields reference these flags by name. Pages with unmet access flags render a 403 result page automatically.

export default (initialState: InitialState) => {
  const { currentUser } = initialState || {};
  const role = currentUser?.role ?? 'viewer';
  const isAdmin = role === 'admin';
  const isOperator = isAdmin || role === 'operator';
  return {
    canManagePlugins: isAdmin,
    canManageConfig: isAdmin,
    canManageBilling: isAdmin,
    canManageMarketplace: isAdmin,
    canManageRoutes: isAdmin,
    canManageConnections: isOperator,
    canManageSessions: isOperator,
    canManageTrafficGuard: isOperator,
    canManageCache: isOperator,
    canViewAudit: true,
    canViewMetrics: true,
  };
};

API Response Envelope & Interceptors

Standard Response Envelope

// All Admin API responses follow this shape
{
  "success": true,
  "data": { /* actual payload */ },
  "message": "OK"
}

// Error responses:
{
  "success": false,
  "data": null,
  "message": "Connection not found"
}

Request Interceptor (app.tsx)

request: {
  requestInterceptors: [
    (config) => {
      const token = localStorage.getItem('draox_token');
      if (token) {
        config.headers = {
          ...config.headers,
          Authorization: `Bearer ${token}`,
        };
      }
      return config;
    },
  ],
  responseInterceptors: [
    (response) => {
      // Unwrap envelope -- UmiJS dataField: 'data'
      // handles success/error routing automatically
      return response;
    },
  ],
}

8. Reusable Components

11 reusable components cover the most frequently recurring UI patterns across the 22 pages. Each component is self-contained with its own directory, TypeScript props interface, and dark-theme styling.

Stat DarkStatisticCard

A dark-themed Card wrapping Ant Design's Statistic component. Accepts title, value, icon, and optional trend indicator (up/down arrow with percentage). Used on Dashboard, Cache, Sessions, and Connections pages to display key metrics at a glance.

Live RealTimeMetricsCard

A variant of DarkStatisticCard that accepts a fetchFn prop and pollingInterval. Internally uses setInterval + useEffect to call the fetch function and update displayed values automatically. Displays a subtle pulse animation while fetching.

Table ConnectionTable

A ProTable pre-configured for connection data. Predefined columns: ID (monospace, copyable), RemoteAddr, Protocol (Tag colored by TCP/UDP/WS/HTTP), ConnectedAt (formatted datetime), State (Badge), and Actions (Disconnect button with ConfirmActionModal). Used on the Connections list page.

Plugin PluginStatusBadge

A color-coded Badge component showing plugin lifecycle state. State colors: active (green), inactive (gray), enabling (orange processing), disabling (orange). Used in ProTable plugin rows and Plugin Detail page header.

Risk IPReputationGauge

A Gauge chart component (0-100 range) with four colored risk zones: green 0-30 (Clean), yellow 30-60 (Suspicious), orange 60-80 (High Risk), red 80-100 (Malicious). Displays a risk label text below the gauge. Used on the Traffic Guard Reputation page with IP search input.

Events EventTimeline

An Ant Design Timeline component rendering server events in chronological order. Each item shows: timestamp, event type Badge (info/warning/error/critical), and description text. Supports auto-scroll to latest, pause toggle, and event type filter dropdown. Used on EventStream full-page and Dashboard summary.

Health HealthStatusBar

A compact horizontal row of colored indicator dots (green = healthy, yellow = degraded, red = down). Each dot represents one server subsystem (TCP, UDP, WS, HTTP, Database, Cache, Plugins). Used on Dashboard and Cache page. Renders subsystem name in tooltip on hover.

Chart BandwidthChart

An Area chart component showing inbound/outbound bandwidth over time. Uses @ant-design/charts Area with dual series, gradient fill, dark background, and orange/accent color scheme. Mock data: 60 data points at 1-second intervals. Used on Dashboard overview.

WS WebSocketIndicator

A small status badge in the ProLayout header showing the global WebSocket connection state. States: Connected (green dot + text), Connecting (yellow pulsing dot), Disconnected (red dot + retry button). Reads state from useModel('global').

Modal ConfirmActionModal

A reusable modal wrapper for dangerous/destructive actions. Renders a warning icon, configurable title, warning description text, and confirm/cancel buttons. The confirm button is styled red/orange and requires a deliberate click. Used by disconnect, session destroy, cache flush, route delete, and ban actions.

Table SearchableIPTable

An Ant Design Table with an IP address search input above it. Columns: IP Address (monospace), Added At (formatted datetime), Reason (if applicable), and Actions (Remove button). Client-side filtering on the IP column. Used for Traffic Guard whitelist and blacklist sub-tables.

UmiJS Mock Convention

All mock files use the export default pattern with an object mapping route strings to handler functions. UmiJS automatically serves these via the dev server — no separate configuration needed. Mock files support hot-reload; changes take effect immediately in the browser without restarting the dev server. 

// Example: mock/connections.ts
import type { Request, Response } from 'express';

const connections = [
  { id: 'conn-001', remoteAddr: '192.168.1.100:52341', protocol: 'WebSocket', state: 'active', ... },
  // ... 7 more entries
];

export default {
  'GET /api/connections': (_req: Request, res: Response) => {
    res.json({ success: true, data: { items: connections, total: connections.length } });
  },
  'GET /api/connections/:id': (req: Request, res: Response) => {
    const conn = connections.find(c => c.id === req.params.id);
    res.json({ success: !!conn, data: conn ?? null });
  },
  'POST /api/connections/:id/disconnect': (_req: Request, res: Response) => {
    res.json({ success: true, message: 'Connection disconnected' });
  },
};

Implementation Details

Barrel Exports

Each language has a barrel file (en-US.ts, vi-VN.ts) that imports and re-exports all four namespace files using spread merging. UmiJS locale plugin consumes these barrel files as the single entry point per locale.

// locales/en-US.ts
import menu from './en-US/menu';
import pages from './en-US/pages';
import component from './en-US/component';
import global from './en-US/global';
export default { ...menu, ...pages, ...component, ...global };

Usage in Pages

All user-facing strings use the useIntl hook or formatMessage utility from @umijs/max. Table column titles, button labels, form field labels, and status messages are all translated.

import { useIntl } from '@umijs/max';

const { formatMessage } = useIntl();
// Usage
const title = formatMessage({ id: 'menu.connections' });
// en-US: "Connections"
// vi-VN: "Kết nối"

Language Switcher

ProLayout's rightContentRender slot includes the UmiJS SelectLang component. Clicking a language calls setLocale(), persists the choice to localStorage, and re-renders all translated strings without a full page reload. Ant Design 5 component labels (date pickers, pagination, form validation) also switch via the locale prop driven by the same setting.

13. Known Limitations & Next Steps

Phase 1+2 delivers a complete scaffold with UI prototypes and mock data. The following items are documented limitations in the current phase, all planned for resolution in Phase 3 (Real API Integration).
Item Status Notes
Traffic Guard CIDR / Regex Ban Not Supported Ban form UI and service layer accept single IPv4/IPv6 only. CIDR ranges and regex patterns require backend API support and frontend input validation changes.
Whitelist / Blacklist Remove Missing API Remove buttons rendered in SearchableIPTable are disabled. Backend DELETE endpoint for whitelist/blacklist entries is not yet implemented in the Admin API.
Whitelist / Blacklist List Missing API IP list data is hardcoded in mock. Backend GET endpoints for listing whitelist and blacklist entries are not yet implemented.
IP Reputation Reset Missing API Reset reputation button is visible but disabled on the Reputation page. Backend endpoint for resetting a reputation score is not yet defined in the Admin API specification.
WebSocket Real-Time Data Mock Only wsManager.ts connects to mock endpoints. All 5 WebSocket streams deliver simulated data. Real WebSocket integration with the backend Admin API WS endpoint is planned for Phase 3.
Real API Integration Phase 3 Replace all mock handlers with real HTTP calls to the Draox Server Admin API (localhost:9100). Requires backend to be running. Error handling, loading states, and pagination need validation against real response shapes.
E2E Test Suite Phase 3 No end-to-end tests in Phase 1+2. Phase 3 will introduce a Playwright or Cypress test suite covering login flow, ProTable data rendering, RBAC page guards, and critical action confirmations.
Build Optimization Phase 3 Current build output is not optimized for production bundle size. Phase 3 work includes: tree-shaking audit for @ant-design/charts, lazy-loading for marketplace and audit pages, and potential icon subset extraction for Ant Design icons.
