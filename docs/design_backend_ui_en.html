1. Overview

The Draox Server Admin Dashboard is built on Ant Design Pro 6, an enterprise-grade React admin framework. The UI provides full visibility and control over all server subsystems — connections, sessions, plugins, traffic guard, metrics, marketplace, billing, audit logs, and more — through a cohesive dark-themed interface with real-time WebSocket updates.

React Ant Design Pro 6

Built on React 18 + UmiJS 4 + Ant Design 5. Provides ProLayout, ProTable, ProForm, ProCard out of the box. Enterprise-grade admin framework with built-in routing, data models, access control, and i18n support.

  • ProLayout for sidebar/header shell
  • ProTable with server-side pagination
  • ProForm for all data entry forms
  • ProCard for content panels and pricing

UI Dark Theme

Custom Ant Design 5 token mapping derived from the server's design palette. Fully consistent with the architecture design reports. Navy + Orange + Green color system.

  • Navy background: #1a1a2e (bg-dark)
  • Orange primary: #e05d10 (--primary)
  • Green accent: #53c28b (--accent)
  • realDark navTheme with custom tokens

WS Real-time Updates

WebSocket integration for all 5 live streams from the Admin API. Auto-reconnect via reconnecting-websocket with exponential backoff. Live metrics, events, guard alerts, connection changes, and plugin lifecycle events.

  • /ws/events — all server events
  • /ws/connections — connection events
  • /ws/plugins — plugin lifecycle events
  • /ws/guard — traffic guard events
  • /ws/metrics — 5-second snapshots

AntD Multi-language & RBAC

Full i18n support for English and Vietnamese via UmiJS locale plugin. Role-based access control (RBAC) with three roles: Admin, Operator, and Viewer. Access factory integrated with UmiJS access plugin.

  • en-US and vi-VN locale files
  • SelectLang component in header
  • 3 RBAC roles with route guards
  • Conditional rendering per role

2. Architecture

The frontend is a standard UmiJS 4 project scaffolded with create-umi. It is organized into config, src/pages, src/services, src/components, src/models, src/locales, and src/utils. The Admin API runs on port 9100 and the dev proxy maps /api to it.

Ring Buffer Implementation

// src/models/metrics.ts
import { useState, useCallback } from 'react';
import type { MetricsSnapshot } from '@/services/metrics';

const MAX_SNAPSHOTS = 60; // 5 minutes at 5s interval

export default function useMetricsModel() {
  const [snapshots, setSnapshots] = useState<MetricsSnapshot[]>([]);
  const [latest, setLatest] = useState<MetricsSnapshot | null>(null);

  const addSnapshot = useCallback((snap: MetricsSnapshot) => {
    setSnapshots(prev => {
      const next = [...prev, snap];
      return next.length > MAX_SNAPSHOTS ? next.slice(-MAX_SNAPSHOTS) : next;
    });
    setLatest(snap);
  }, []);

  return { snapshots, latest, addSnapshot };
}

WebSocket Manager Design

// src/services/wsManager.ts
import ReconnectingWebSocket from 'reconnecting-websocket';

type StreamName = 'events' | 'connections' | 'plugins' | 'guard' | 'metrics';
type Listener = (data: unknown) => void;

const WS_PATHS: Record<StreamName, string> = {
  events:      '/ws/events',
  connections: '/ws/connections',
  plugins:     '/ws/plugins',
  guard:       '/ws/guard',
  metrics:     '/ws/metrics',
};

class WsManager {
  private sockets: Partial<Record<StreamName, ReconnectingWebSocket>> = {};
  private listeners: Partial<Record<StreamName, Set<Listener>>> = {};

  connect(stream: StreamName): void {
    if (this.sockets[stream]) return;

    const token = localStorage.getItem('draox_token');
    const base  = window.location.origin.replace(/^http/, 'ws');
    const url   = `${base}${WS_PATHS[stream]}?token=${token}`;

    const ws = new ReconnectingWebSocket(url, [], {
      maxRetries: Infinity,
      minReconnectionDelay: 1000,
      maxReconnectionDelay: 30000,
      reconnectionDelayGrowFactor: 1.5,
    });

    ws.onmessage = (evt) => {
      try {
        const data = JSON.parse(evt.data as string);
        this.listeners[stream]?.forEach(fn => fn(data));
      } catch { /* ignore malformed frames */ }
    };

    this.sockets[stream] = ws;
  }

  subscribe(stream: StreamName, listener: Listener): () => void {
    if (!this.listeners[stream]) this.listeners[stream] = new Set();
    this.listeners[stream]!.add(listener);
    this.connect(stream);

    // Return unsubscribe function
    return () => {
      this.listeners[stream]?.delete(listener);
      // Auto-disconnect if no more listeners
      if (this.listeners[stream]?.size === 0) {
        this.disconnect(stream);
      }
    };
  }

  disconnect(stream: StreamName): void {
    this.sockets[stream]?.close();
    delete this.sockets[stream];
    delete this.listeners[stream];
  }

  disconnectAll(): void {
    (Object.keys(this.sockets) as StreamName[]).forEach(s => this.disconnect(s));
  }
}

export const wsManager = new WsManager();

14. Service Layer

All API calls are centralized in 13 typed service files under src/services/. Each file exports async functions that wrap the UmiJS request() utility with TypeScript interfaces matching backend response types.

HTTP Client Configuration (src/app.tsx)

// src/app.tsx — Runtime configuration
export const request: RequestConfig = {
  baseURL: '/api',
  timeout: 30000,

  // Auth interceptor: inject JWT token
  requestInterceptors: [
    (config) => {
      const token = localStorage.getItem('draox_token');
      if (token) {
        config.headers = { ...config.headers, Authorization: `Bearer ${token}` };
      }
      return config;
    },
  ],

  // Response interceptor: unwrap { success, data, message } envelope
  responseInterceptors: [
    (response) => {
      const { data } = response;
      if (data?.success === false) {
        throw new Error(data.message ?? 'Request failed');
      }
      return data?.data ?? data;
    },
  ],

  // Error handler
  errorConfig: {
    errorHandler(error: RequestError) {
      if (error.response?.status === 401) {
        localStorage.removeItem('draox_token');
        window.location.href = '/login';
        return;
      }
      if (error.response?.status === 429) {
        message.error('Rate limit exceeded. Please slow down.');
        return;
      }
      message.error(error.message ?? 'An error occurred');
    },
  },
};

Design Principles

Consistency First

All colors, typography, spacing, and interaction patterns derive from the same design token system as the architecture docs. No ad-hoc values — everything maps to a CSS variable or Ant Design token.

Real-time by Default

Pages that benefit from live data connect to a WebSocket stream on mount and disconnect on unmount. The WsManager handles auto-reconnect transparently — pages never need to handle WS lifecycle manually.

RBAC Everywhere

Access control is enforced at three levels: route guards (page-level), component rendering (action buttons), and API layer (backend always re-validates). The frontend RBAC is a UX layer, not a security boundary.

TypeScript Throughout

All service responses, component props, model state, and event payloads are fully typed. TypeScript interfaces mirror the backend Rust types (manually kept in sync or generated via openapi-typescript from the Swagger spec).
